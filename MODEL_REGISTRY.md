# Model Registry

This document lists every ML model used or trained by the BRIGHT pipeline,
where to find it, what it covers, and how to retrain it.

### Known limitations (all models)

- **Trained exclusively on synthetic data** generated by an LLM pipeline
  (`bright_models/generated_data/`). Real-world performance is substantially
  lower than the synthetic benchmark (macro F1 ≈ 0.18 rules-only on the
  9-document held-out test set; see
  `plans_and_analyses/RefiningTheFinalPipeline.md` for a full diagnosis).
- **Vocabulary-unconstrained fields** (`diag_histologique`, `diag_integre`,
  `tumeur_position`) are prone to hallucinating nearby tokens (e.g. a
  surgeon's name extracted as tumour position). A vocabulary validation gate
  is in place for constrained fields (see `src/extraction/pipeline.py`,
  `_hf_value_is_plausible`), but free-text fields are not gated.
- Models are CPU-only by default (`CUDA_VISIBLE_DEVICES=""` in setup scripts).

## Retraining

### Prerequisites

- Access to the HuggingFace Hub account `raphael-r` (or fork to your own account
  and update `HUB_PREFIX` in `bright_models/push_to_hub.py` and
  `HUB_PREFIX` in `src/extraction/hf_extractor.py` line 61).
- A GPU instance (Google Colab is documented in `bright_models/`).
- The `HF_TOKEN` environment variable set to a HuggingFace write token.

### Steps

1. **Generate or extend the training dataset**
   See `bright_models/generated_data/pipeline/` for the 4-step LLM generation
   pipeline. Current dataset: `generated_data/data/generated_dataset_extended.jsonl`.

2. **Train per-group models**
   See `bright_models/eds_bright/` for training scripts.
   ```bash
   cd bright_models
   python training.py --group diagnosis
   ```

3. **Evaluate**
   ```bash
   python bright_models/eds_bright/scripts/evaluate.py --group diagnosis
   ```

4. **Push to HuggingFace Hub**
   ```bash
   python bright_models/push_to_hub.py
   ```

5. **Update this registry**, add training date, dataset version, and F1 scores.
